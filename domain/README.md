# 领域层目录结构约定（DDD）

本目录按“领域 → 子领域 → 界限上下文（Bounded Context）”三层组织代码。
在开始开发之前，界限上下文可能尚未完全明确，因此建议：
- 先确认领域与子领域划分，再在每个界限上下文内创建聚合、仓储接口、领域服务等；
- 未归类到具体上下文的通用基类、基元放在 `common/` 下；
- 避免在 `domain/` 根目录下直接放置聚合/实体等未分类内容。

推荐结构示例：

```
DOMAIN_ROOT/
├── 领域A/
│   ├── 子领域A1/
│   │   └── 上下文A1-1/
│   │       ├── aggregates/
│   │       │   ├── 订单/
│   │       │   │   ├── 订单.py            # 聚合根
│   │       │   │   ├── entities/          # 聚合内实体
│   │       │   │   └── value_objects/     # 聚合内值对象
│   │       │   └── 客户/
│   │       ├── repositories/              # 仓储接口
│   │       │   ├── 订单_repository.py
│   │       │   ├── 客户_repository.py
│   │       │   └── base_repository.py     # 可按上下文扩展的基础接口
│   │       ├── services/                  # 领域服务
│   │       ├── events/                    # 领域事件
│   │       ├── policies/                  # 领域策略
│   │       └── exceptions/                # 领域异常
│   └── 子领域A2/
├── 领域B/
│   └── ...
```
